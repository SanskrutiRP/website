Master Node Services

These services facilitate operations pertaining to the moderator role.

The services can only be called from the API service and are shielded from being accessible publicly.

They directly interact with the blockchain smart contract and have access to the wallet private key having moderator role in the NetSepio Smart Contract.

Functionalities:

Web Scraping

Machine Learning model for sentiment analysis

Smart Contract Events Logging

Give Voter Role to new user wallets

API Gateway

NetSepio API Gateway provides interaction with the blockchain smart contract and data retrieval supported by an authentication middleware.

Endpoints:

/api/v1.0/authenticate - POST - Issue PASETO against a signed message (personal.sign) containing the Unique authId within the time limit. Also creates basic profile details

/api/v1.0/verify - POST - Verify the user’s twitter account against a specific tweet - For getting a verified status on their profile*

/api/v1.0/profile - PATCH - Users can update their basic profile details.

/api/v1.0/registertovote - POST - Request voting rights to submit votes

/api/v1.0/artifact - POST - Store NFT metadata on IPFS

/api/v1.0/vote - POST - Submit vote against a unique websiteURL along with storing the NFT metadata

/api/v1.0/vote - DELETE - Delete vote against a unique websiteURL*

/api/v1.0/getVote - GET - Get vote details against a websiteURL

/api/v1.0/websiteStatus - GET - Get details about a websiteURL taking account of all the votes registered towards the website and domain

Smart Contract

Functions

Submit Vote / Mint Function:
  • Must have voter role to call the function directly using the user wallet
  • Accepts domainName, shortURL, urlHash, category, tag
  • Creates a struct for the review with the tokenId & owner so that only 1 NFT is minted in lieu of the urlHash
  • Event Log
Delegate Vote Submission / Mint on behalf of the voter:
  • Must have moderator role to call this function to create the NFT and make the voter its owner
  • Accepts domainName, shortURL, urlHash, category, tag and owner
  • Creates a struct for the review with the tokenId & owner so that only 1 NFT is minted in lieu of the urlHash
  • Event Log
Find token details by voter address and urlHash
Delete Function:
  • Only token owner or approved can burn the token
  • Burn the Token - Delete the Review
Update the category
